1. 引言

在单层网络中我们可以用邻接矩阵来表示有无方向、有无权重的网络结构。但是一旦拓展到多层网络中，矩阵就显得很无力。参考这篇文献，此博文通过张量的形式来表达多层网络以及相关的网络分析。
为了更好表述，我们对数学符号进行了规定。对于无向网络，我们使用大写的A，而对于加权网络我们使用大写的W。另外，为验证公式的正确性，我们假设网络有N个节点，并在python中定义无向无权网络为A_uu，无向加权为A_uw, 有向无权网络为A_du, 有向加权网络为A_dw。然后，随机模拟生成这4个单层网络。具体如下：

import numpy as np
import math
# 定义网络节点的个数
N = 10
# 节点i,j存在边的概率
p = 0.4
# 权重大小
w = 5
# 创建网络
def createNetwork(type, N=10, p=0.4, w=None):
  A = np.zeros((N, N))
  for i in range(N):
    for j in range(N):
      if i != j and np.random.rand() < p:
        if type == 'dw':
          A[i][j] = np.random.randint(w)
        elif type == 'uw':
          A[i][j] = A[j][i] = np.random.randint(w)
        elif type == 'du':
          A[i][j] = 1
        elif type == 'uu':
          A[i][j] = A[j][i] = 1
        else:
          assert 0, "the type should be 'uu', 'uw', 'du', 'dw'"
  return A

Note that for the network without self-loop, $a_{ii}=0$.

2. 度中心性（Degree centrality）

the degree centrality vector: $$ k_{\beta} = A_{\beta}^{\alpha}u_{\alpha} $$ where the $u_{\alpha}={1,1,…,1}^T$ whose components are all equal to 1, and then we could directly calculate the degree centrality of node $n_i$: $$ k(i) = k_{\beta}e^{\beta}(i) $$

# the dot product
N = 10
A_uu = createNetwork('uu', N=10, p=0.4)
# the covariant and contravariant vector
u_cov = np.ones(N)
u_con = np.ones(N)
# the degree vector
k = np.einsum('ij,i->j',A_uu, u_cov)
# For example, we calculate the degree centrality of 1-th node
e_i_con = np.zeros(N)
e_i_cov = np.zeros(N)
e_i_con[0] = e_i_cov[0]= 1
k_i = np.einsum('i,i',k, e_i_con)
print('i = 1, k_i = ', k_i)

Note that the i = $\alpha$, and j = $\beta$, correspondingly. and if you want to standardize, the above equation should divided by (N-1).

Further, according to the above equations we also can extend the situation to a weighted network to the strength centrality(vector):
$$ s_{\beta} = W_{\beta}^{\alpha}u_{\alpha} $$

A_uw = createNetwork('uw', N=10, p=0.4, w=5)
# the strength vector
s = np.einsum('ij,i->j', A_uw, u_cov)
print('s = ', s)

Besides, we can calculate the mean degree, the second moment of the degree, and the variance of the degree.
the mean degree: $$ \langle k \rangle = (U_{\rho}^{\rho})^{-1}k_{\beta}u^{\beta}$$ the second moment of the degree: $$ \langle k^2 \rangle = (U_{\rho}^{\rho})^{-1}k_{\beta}k^{\beta}$$ the variance of the degree: $$ var(k) = (U_{\rho}^{\rho})^{-1}k_{\beta}k^{\beta} - (U_{\rho}^{\rho})^{-2}k_{\alpha}k_{\beta}U^{\alpha\beta}$$

Note that the $U_{\rho}^{\rho}$ is a scalar, that the total number of nodes, and it is equal to the trace of $U_{\alpha}^{\beta}$, and $U_{\alpha}^{\beta}$ is the second-order tensor whose elements are all equal to 1, $U_{\alpha}^{\beta}=u_{\alpha}u^{\beta}$.

# 向量的外积
U = np.einsum('i,j->ij', u_cov, u_con) # 外积(张量积)
# the trace of U
u = np.einsum('ii', U) # 矩阵对角元素相加之和
# the mean degree
k_mean = np.einsum('i,i', k, u_con) / u
# the second moment of the degree
k_2_moment = np.einsum('i,i', k, k) / u
# the variance of the degree
k_var = np.einsum('i,i', k, k) / u - np.einsum('ij,ij',np.einsum('i,j->ij', k, k), U) / u**2

Finally, we consider the directed network. e.g. the in-degree centrality and out-degree centrality are represented using different tensor products.
the in-degree centrality: $$ k_{\beta} = A_{\beta}^{\alpha}u_{\alpha} $$ the out-degree centrality: $$ k^{\alpha} = A_{\beta}^{\alpha}u^{\beta} $$

A_du = createNetwork('du', N=10, p=0.4)
# the in-degree centrality
k_in = np.einsum('ij,i->j', A_du, u_cov)
# the out-degree centrality
k_out = np.einsum('ij,j->i', A_du, u_con)

We then recover the usual definitions for directed networks. For example, the in-degree centrality of node $n_i$: $$k_{in}(i) = A_{\beta}^{\alpha}u_{\alpha}e^{\beta}(i)$$

k_in_i = np.einsum('i,i', k_in, e_i_con) # 内积(点乘)

At last, we also can calculate the degree centrality of a network and standardization: $$ k(A_{\beta}^{\alpha}) = [(N-1)(N-2)]^{-1}(p_{\alpha} - k^{\alpha}) $$

Note that $p_{\alpha}$whose components are all equal to the maximum of $k_{\alpha}$. (N-1)(N-2) is the maximum possible connection.

p = np.ones(N) * np.max(k)
k_network = np.einsum('i->', p - k) / ((N-1) * (N-2))

3.聚集系数（Clustering coefficients）

The clustering coefficients are useful measures of transitivity in a network. It includes two typies that the local clustering coefficient of a node $n_i$ and the global clustering coefficient.
For unweighted and undirected networks, the local clustering coefficient of a node $n_i$ is defined as the number of existing edges among the set of its neighboring nodes divided by the total number of possible connections between them. $$ c(i) = {\mathcal{N}}_i / N_i(N_i - 1)/2\tag{3.1} $$

Note that ${\mathcal{N}}_i$ indicate the number of existing edges among the set of its neighboring nodes, and $N_i$ is the total number of possible connections between them.

One can define the local clustering coefficient by dividing the number of three-cycles by the number of three-cycles in a network for which the neighborhood of the node $n_i$ is completely connected. We thereby obtain the formula: $$ c(A_{\beta}^{\alpha},i) = \frac{A_{\rho}^{\alpha}A_{\sigma}^{\rho}A_{\beta}^{\sigma}e_{\alpha}(i)e^{\beta}(i)}{A_{\rho}^{\alpha}F_{\sigma}^{\rho}A_{\beta}^{\sigma}e_{\alpha}(i)e^{\beta}(i)}\tag{3.2} $$ where $$F_{\sigma}^{\rho} = U_{\sigma}^{\rho} - {\delta}_{\sigma}^{\rho} $$

F = U - np.eye(N)

is the adjacency tensor corresponding to a network that includes all edges except for self-loops.

c_i = np.einsum('ij,jk,kl,i,l', A_uu, A_uu, A_uu, e_i_cov, e_i_con) /
      np.einsum('ij,jk,kl,i,l', A_uu, F, A_uu, e_i_cov, e_i_con)

The global clustering coefficient of a network: $$ c(A_{\beta}^{\alpha}) = \frac{A_{\rho}^{\alpha}A_{\sigma}^{\rho}A_{\beta}^{\sigma}}{A_{\rho}^{\alpha}F_{\sigma}^{\rho}A_{\beta}^{\sigma}}\tag{3.3} $$

c = np.einsum('ij,jk,kl', A_uu, A_uu, A_uu) / np.einsum('ij,jk,kl', A_uu, F, A_uu)

And we can consider a weighted network. if the $W_{\beta}^{\alpha}$ corresponds to **the weighted adjacency tensor normalized such that each element of the tensor lies in the interval [0, 1]**, we can define a clustering coefficient in a weighted network without any changes to Eqs. (3.2) and (3.3). Otherwise, the weighted clustering coefficients should be defined as follow: $$ c(W_{\beta}^{\alpha}) = \mathcal{N}^{-1}\frac{W_{\rho}^{\alpha}W_{\sigma}^{\rho}W_{\beta}^{\sigma}}{W_{\rho}^{\alpha}F_{\sigma}^{\rho}W_{\beta}^{\sigma}}\tag{3.4} $$

Note that Let $\mathcal{N}$ be a real number that can be used to rescale the elements of the tensor. e.g. it can come from the maximum (so that $\mathcal{N}=\max_{\alpha,\beta}{W_{\beta}^{\alpha}}$).

max_w = np.max(A_uw) # the maximum of W
c = np.einsum('ij,jk,kl', A_uw, A_uw, A_uw) / np.einsum('ij,jk,kl', A_uw, F, A_uw) / max_w

The same argument applies in the case of the local clustering coefficient for weighted networks.

4.特征向量中心性和Katz中心性（Eigenvector and Katz centralities）

In our tensorial formulation, the eigenvector centrality vector is a solution of the tensorial equation: $$W_{\beta}^{\alpha}v_{\alpha}=\lambda_{1}v_{\beta}\tag{4.1}$$ $$W_{\beta}^{\alpha}v^{\beta}=\lambda_{1}v^{\alpha}\tag{4.2}$$

and $v_{\alpha}e^{\alpha}(i)$ gives the eigenvector centrality of node $n_i$. For directed networks, there are two leading eigenvectors, and one needs to take into account the difference between Eq.(4.1) and its contravariant counterpart Eq.(4.2).

# in-degree
w, vr = np.linalg.eig(A_du)
# out-degree; see detail in networkx.eigenvector_centrality.[Notes]
w, vl = np.linalg.eig(A_du.T)
ec = vr[:,np.argmax(w)].astype(np.float64)
ec_i = np.einsum('i,i', ec, e_i_con)

However, nodes with only outgoing edges have an eigenvector centrality of 0 if the above definition is adopted. For this, we need the Katz centralities. the leading-eigenvector solution of the eigenvalue $\mathbf{v}=a\mathbf{Av}+b\mathbf{1}$, where 1 is a vector in which each entry is a 1, and b=1. The parameter b controls the initial centrality and $$a<\frac{1}{\lambda_1}$$ Using tensorial notation, we obtain $$v_{\beta}=(\delta_{\beta}^{\alpha}-aW_{\beta}^{\alpha})^{-1}u_{\alpha}\tag{4.3}$$ $$v^{\alpha}=(\delta_{\beta}^{\alpha}-aW_{\beta}^{\alpha})^{-1}u^{\beta}\tag{4.4}$$

To calculate Katz centrality from Eq. (4.3), we need to calculate the tensor inverse $T_{\beta}^{\alpha}$, which satisfies the equation, $T_{\beta}^{\alpha}(\delta_{\sigma}^{\beta}-aW_{\sigma}^{\beta})=\sigma_{\delta}^{\alpha}$.where $\mathcal{K}=W_{\beta}^{\alpha}U_{\alpha}^{\beta}$ .To consider a general null model, we write $B_{\beta}^{\alpha}=W_{\beta}^{\alpha}-P_{\beta}^{\alpha}$, where $P_{\beta}^{\alpha}$ is a tensor that encodes the random connections against which one compares a network’s actual connections.

# alpha = 0.1, b = 1. see detail in networkx.centrality.katz_centrality
T = np.linalg.inv(np.eys(N) - 0.1 * A_du)
vr = np.einsum('ij, i', T, u_cov)
vl = np.einsum('ij, j', T, u_con)

5.社区发现与模块度（Community Detection and Modularity）

Find communities in network using Clauset-Newman-Moore greedy modularity maximization. This method currently does not consider edge weights. Greedy modularity maximization begins with each node in its own community and joins the pair of communities that most increases modularity until no such pair exists(see detail in networkx.algorithms.community). We define a tensor $S_a^{\alpha}$ in $\mathbb{R}^{N \times M}$, where $\alpha$ indexes nodes and $\mathcal{a}$ indexes the communities. The value of a component of $S_a^{\alpha}$ is defined to be 1 when a node belongs to a particular community and 0 when it does not. The modularity of a network partition is given by the scalar: $$Q = \frac{1}{\mathcal{K}}S_{\alpha}^{a}B_{\beta}^{\alpha}S_a^{\beta}\tag{5.1}$$ where $\mathcal{K}=W_{\beta}^{\alpha}U_{\alpha}^{\beta}$ .To consider a general null model, we write $B_{\beta}^{\alpha}=W_{\beta}^{\alpha}-P_{\beta}^{\alpha}$, where $P_{\beta}^{\alpha}$ is a tensor that encodes the random connections against which one compares a network’s actual connections.

from networkx.algorithms.community import greedy_modularity_communities

6.网络扩散过程————随机游走（Diffusion and random walks）

A random walk is the simplest dynamical process that can occur on a monoplex network, and random walks can be used to approximate other types of diffusion.Let $x_{\alpha}(t)$ denote a state vector of nodes at time t. The diffusion equation is $$ \frac{dx_{\beta}(t)}{dt} = D\left[W_{\beta}^{\alpha}x_{\alpha}(t)-W_{\gamma}^{\alpha}u_{\alpha}e^{\gamma}(\beta)x_{\beta}(t)\right]\tag{6.1} $$ where D is a diffusion constant, Recall that $s_{\gamma}=W_{\gamma}^{\alpha}u_{\alpha}$ is the strength vector and that $s_{\gamma}e^{\gamma}(\beta)x_{\beta}(t)=s_{\gamma}e^{\gamma} \times \delta_{\beta}^{\alpha} x_{\alpha}(t)$. We obtain the following covariant diffusion law on monoplex networks: $$\frac{dx_{\beta}(t)}{dt} = -DL_{\beta}^{\alpha}x_{\alpha}(\alpha)\tag{6.2}$$ where $L_{\beta}^{\alpha}=W_{\gamma}^{\eta}u_{\eta}e^{\gamma}(\beta)\delta_{\beta}^{\alpha}-W_{\beta}^{\alpha}$ is the combinatorial Laplacian tensor. The solution of Eq. (6.2) is $x_{\beta}(t)=x_{\alpha}(0)e^{-DL_{\beta}^{\alpha}t}$.

# a diffusion constant
D = 0.1
# at time t = 5
t = 5
# the initial state
x0 = np.random.randint(0, 2, 10)
# the degree matrix
dm = np.zeros((N,N))
for i in range(N): e_i_con = np.zeros(N); e_i_con[i] = 1; dm[i][i] = np.einsum('ij,i,j', A_du, u_cov, e_i_con)
L = dm - A_du
x_t = np.einsum('i, ij', x0, np.exp(-D * t * L))

注：拉普拉斯矩阵（张量）L: L = D - A, 其中D为网络的度矩阵(degree matrix)，A为网络的邻接矩阵。度矩阵D是对角阵，对角上的元素为各个顶点（ps.在数据结构链表中称为节点，图中称为顶点）的度。有向图中，顶点v的度分为出度和入度，即从顶点v出去的有向边的数量和进入顶点v的有向边的数量（百度百科/维基百科中认为只需要其中一个，文献中也是如此。但是，下面参考CSDN中却是相加！？）。

——NOTE——

如何使用einsum?

einsum中的字符串是什么意思？想象’ij,jk->ik'在箭头->处分成两部分。左侧部分标记输入数组的轴：‘ij'标记A和’jk'标记B。字符串的右侧部分用字母’ik'标记单个输出数组的轴。也就是说，我们正在传入两个二维数组，获取一个新的二维数组。
如果我们省略箭头’->’，NumPy会将只出现一次的标签按照字母顺序排列（因此，实际上’ij,jk->ik'相当于’ij,jk’）。如果我们想控制输出的样子，我们可以自己选择输出标签的顺序。例如，‘ij,jk->ki'为矩阵乘法的转置。

要了解输出数组的计算方法，请记住以下三个规则：

  • 在输入数组中重复的字母意味着值沿这些轴相乘。乘积结果为输出数组的值。
    在本例中,我们使用字母j两次：A和B各一次。这意味着我们将A每一行与B每列相乘。这只在标记为j的轴在两个数组中的长度相同（或者任一数组长度为1）时才有效。

  • 输出中省略的字母意味着沿该轴的值将相加。
    在这里，j不包含在输出数组的标签中。通过累加的方式将它从轴上除去，最终数组中的维数减少1。如果输出是’ijk’，我们得到的结果是3x3x3数组（如果我们不提供输出标签，只写箭头，则对整个数组求和）。

  • 我们可以按照我们喜欢的任何顺序返回未没进行累加的轴。

标准化与归一化

“标准化"和"归一化"指代四种Feature scaling(特征缩放)方法：

  • Rescaling

$$x_i^{'} = \frac{x_i - min(x)}{max(x)-min(x)}$$

  • Mean normalization

$$x_i^{'} = \frac{x_i - mean(x)}{max(x)-min(x)}$$

  • Standardization

$$x_i^{'} = \frac{x_i - \overline{x}}{\sigma(x)}$$

  • Scaling to unit length

$$x_i^{'} = \frac{x_i}{\Vert x \Vert}$$

社区发现与模块度

社区发现领域有两种比较经典的模型，一类就是基于Null model的模型（代表工作是Modularity），一类是基于生成式模型的方法（代表工作是Block model和Pisson model等）。Newman也已经证明了Null model和Block model之间的联系，这里不详细讲了。

关于Null model，其实思想很简单，一句话就是“对比你划分的结果和随机网络的差异”。怎么理解呢？其实Null model可以理解为一个参照系的原点，这个原点选择了随机网络，也就是没有任何结构性的网络。那么对于一个网络你划分得到的结果跟这个参照系的原点差别越大说明你这个网络上的这个划分的结果结构性越好。因此，可以这么说，Modularity并不是在进行社区发现，而是在进行网络结构性的评价。社区只是反映这种结构性质的一个表现。
